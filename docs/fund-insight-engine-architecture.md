# Fund Insight Engine——投资思维平台架构设计

> **愿景**：不是给投资人一个基金筛选器，而是给投资人一个用自己思维方式构建基金组合的思想平台。
>
> 投资的本质是"我相信什么"。这个平台让投资人从认知出发，经过数据验证、产业链穿透、估值检查、组合构建，最终得到一个匹配自己认知的配置方案。
>
> 创建时间：2026-07-08

---

## 一、为什么做这个

### 1.1 现有基金工具的根本问题

市面上的基金工具——无论是天天基金、蚂蚁财富、还是机构的投研系统——都在做同一件事：

```
给你一堆基金 → 帮你筛选/排序 → "这是最好的基金"
```

这回答不了投资人真正的问题：**"我相信X，哪些基金匹配我的认知？"**

会议中导师说得很清楚：

> "你给我一个组合，你就给我一个基金的一个清单，给我一个配置的东西。我还是不知道你感觉。"

问题的本质是：**工具给的是数据，投资人要的是认知匹配。**

### 1.2 会议揭示的投研真相

会议中导师分享了自己的投资组合，基于四个认知：

| 认知 | 投什么 | 为什么 |
|---|---|---|
| AI是巨大生产力变革 | 沪港通AI基础设施基金 | 相信生产力爆炸 |
| 创新药被错杀 | 创新药基金 | 相信被市场低估 |
| 信任付老师 | 付老师管理的产品 | 相信人的能力 |
| 龙头集中 | 沪深300 | 相信中国核心资产 |

他说："我所有的这些东西实际上是基于我的认知来构建。"

这就是投资的真相：**先有认知，再有组合。** 不是先看数据再决定投什么，而是先知道自己相信什么，再找匹配的工具。

### 1.3 这个平台要解决什么

```
投资人的困境：
  "我感觉AI是未来" → 但不知道哪些基金真的投AI
  "我感觉创新药被低估" → 但不知道怎么验证这个判断
  "我选了3只AI基金" → 但不知道它们是不是买同一批股票
  "我买了" → 但不知道估值贵不贵、该配多少

这个平台要做的：
  把"我感觉" → 变成"我认知"（结构化）
  把"我认知" → 用数据验证（正反证据）
  把"我验证" → 穿透到底层资产（不是基金名称）
  把"我穿透" → 检查估值（贵不贵）
  把"我检查" → 构建组合（匹配认知+风险可控）
```

## 二、思想架构：从认知到组合的五步法

### 2.1 核心理念

```
投资不是"选最好的基金"，而是"找到匹配我认知的组合"
```

这个理念决定了平台的交互逻辑：**不是从基金出发，而是从认知出发。**

### 2.2 五步投资思维法

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  第一步：认知输入                                           │
│                                                            │
│  "我相信什么？"                                            │
│  不是选基金，是表达投资判断                                 │
│  ┌──────────────────────────────────────────┐             │
│  │ 认知 = 判断 + 逻辑链 + 产业链 + 估值容忍度  │             │
│  └──────────────────────────────────────────┘             │
│                      ↓                                     │
│  第二步：认知验证                                           │
│                                                            │
│  "我的认知靠谱吗？"                                        │
│  不是替代判断，是提供正反证据                               │
│  ┌──────────────────────────────────────────┐             │
│  │ ✅ 支持证据：利润增速、ROE、收入确认       │             │
│  │ ⚠️ 反面证据：估值分位、拥挤度、周期风险    │             │
│  │ → 判断：认知有效但已过度定价               │             │
│  └──────────────────────────────────────────┘             │
│                      ↓                                     │
│  第三步：资产穿透                                           │
│                                                            │
│  "我到底在买什么？"                                        │
│  不是看基金名称，是穿透到底层股票                           │
│  ┌──────────────────────────────────────────┐             │
│  │ 基金是壳子，底层是股票组合                  │             │
│  │ 产业链穿透：光模块/PCB/芯片/服务器         │             │
│  │ 每只基金买了什么环节的什么股票              │             │
│  └──────────────────────────────────────────┘             │
│                      ↓                                     │
│  第四步：估值检查                                           │
│                                                            │
│  "贵不贵？能买吗？"                                        │
│  不是只看PE绝对值，是三维估值判断                           │
│  ┌──────────────────────────────────────────┐             │
│  │ 历史分位：当前估值在过去什么位置            │             │
│  │ PEG：增速能不能支撑估值                    │             │
│  │ price in：已定价了多少增长                 │             │
│  │ → 热爱产品 ≠ 买股票，估值太贵要控制仓位    │             │
│  └──────────────────────────────────────────┘             │
│                      ↓                                     │
│  第五步：组合构建                                           │
│                                                            │
│  "怎么配？配多少？"                                        │
│  不是选TOP3各配8%，是认知匹配+风险控制                     │
│  ┌──────────────────────────────────────────┐             │
│  │ 匹配度 → 选哪些基金                        │             │
│  │ 估值约束 → 每个配多少                      │             │
│  │ 重叠度+相关性 → 避免集中                   │             │
│  │ 防守仓位 → 自动对冲                        │             │
│  │ → 一份匹配认知的配置方案                   │             │
│  └──────────────────────────────────────────┘             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 2.3 这个方法论的核心原则

1. **认知优先**：从"我相信什么"出发，不从"有什么基金"出发
2. **穿透壳子**：基金是壳子，核心是底层资产
3. **估值是硬约束**：热爱产品 ≠ 买股票
4. **提供证据不替代判断**：信息矛盾时靠"相信"
5. **组合思维**：不是选最好的基金，是选最匹配认知且风险可控的组合
6. **动态跟踪**：认知在变、估值在变、持仓在变

### 3.2 各层职责

#### 前端交互层

投资人的操作入口。不是传统的"筛选基金"界面，而是"表达认知"界面。

| 页面 | 做什么 | 核心交互 |
|---|---|---|
| 认知输入页 | 表达"我相信什么" | 文本输入 + 主题选择 + 产业链环节 + 估值容忍度 |
| 验证结果页 | 看认知的证据 | 支持证据/反面证据/估值定价/最终判断 |
| 基金穿透页 | 看匹配的基金 | 产业链分布 + 持仓个股 + 估值分位 + 持仓趋势 |
| 组合方案页 | 看配置方案 | 基金清单 + 权重 + 重叠度 + 相关性 + 风险指标 |

#### 认知引擎层

平台的核心。把投资人的认知转化为配置方案。

```
backend/app/cognition/
├── __init__.py
├── input.py           # 认知输入模型
├── validator.py       # 认知验证引擎
├── asset_mapper.py    # 产业链穿透
├── valuation_gate.py  # 估值门禁
├── portfolio_builder.py # 组合构建
├── theme_registry.py  # 主题/产业链映射库
└── persistence.py     # 认知组合持久化
```

#### 标签引擎层（现有）

认知引擎的证据底座。现有的标签引擎不废弃，角色从"产品出口"变为"证据提供者"。

| 标签引擎能力 | 在认知引擎中的角色 |
|---|---|
| 基金持仓数据 | 产业链穿透的数据源 |
| 股票因子（PE/PB/ROE/分位） | 估值门禁的数据源 |
| 行业映射 | 产业链匹配的数据源 |
| 收益风险标签 | 组合风险约束的输入 |
| 基准解析 | 相对表现的验证工具 |
| NAV日收益 | 基金相关性计算 |
| 多期持仓 | 持仓变化趋势分析 |

#### 数据层（现有）

所有原始数据的存储。不做改动。

## 四、认知引擎核心模型设计

### 4.1 认知输入模型

```python
class CognitionInput:
    """投资人认知输入——不是一个关键词，是一套投资逻辑"""

    # 核心判断：你相信什么
    belief: str
    # 例: "AI基础设施会先于应用层变现"

    # 认知类型
    cognition_type: str  # theme / sector / stock / style / manager

    # 逻辑链：为什么相信（每条可被数据验证）
    logic_chain: list[str]
    # 例: [
    #   "AI服务器收入确认先于大模型商业化",
    #   "基础设施公司已确认大量收入",
    #   "应用层仍在亏损"
    # ]

    # 产业链环节：具体投哪个环节
    chain_focus: ChainFocus
    # primary: "AI算力基础设施"
    # secondary: "AI芯片设计"
    # exclude: ["AI应用层"]

    # 估值容忍度：能接受多贵
    valuation_tolerance: ValuationTolerance
    # max_pe: 80
    # max_valuation_percentile: 85
    # require_margin_of_safety: false

    # 信心与时间
    conviction: str  # low / medium / high
    time_horizon: str  # short / mid / long
    risk_tolerance: str  # conservative / moderate / aggressive
```

### 4.2 认知验证模型

```python
class CognitionValidation:
    """认知验证结果——不替代判断，只提供证据"""

    supporting_evidence: list[Evidence]
    # ✅ 利润增速 129%，基本面强劲
    # ✅ 加权ROE 21.4%，盈利能力强
    # ✅ PEG 0.83，增速能支撑估值

    opposing_evidence: list[Evidence]
    # ⚠️ 估值分位 94%，处于历史高位
    # ⚠️ 持仓集中度高，拥挤度风险

    valuation_assessment: ValuationAssessment
    # 当前定价程度: 已price in约2年高增长
    # 安全边际: 低
    # 赔率: 不佳（认知对但估值贵）

    verdict: str  # validated / partially_validated / unvalidated
    # "认知有效但已过度定价"
```

### 4.3 产业链穿透模型

```python
class AssetMapping:
    """认知 → 底层资产 → 基金的穿透结果"""

    chain_breakdown: dict[str, ChainLinkMatch]
    # "光模块/连接": {
    #     "stocks": ["中际旭创", "天孚通信", "新易盛"],
    #     "fund_match": {"000522": 27.7%, "000063": 15.2%}
    # }
    # "PCB/电子元件": {
    #     "stocks": ["沪电股份", "深南电路", "生益科技"],
    #     "fund_match": {"000522": 63.7%, "000063": 22.1%}
    # }

    fund_matches: list[FundMatch]
    # 每只基金的匹配度 + 产业链分布 + 匹配个股

    holding_trend: dict[str, TrendData]
    # 每只基金对该认知的持仓变化趋势
    # "000522": {2025Q1: 40.7% → 2025Q4: 83.2%, trend: "increasing"}
```

### 4.4 估值门禁模型

```python
class ValuationGate:
    """估值三维判断——不只是PE绝对值"""

    # 维度1：历史分位
    valuation_percentile: float  # 0-1
    # 当前估值在过去什么位置

    # 维度2：PEG
    peg: float
    # PE ÷ 利润增速，判断增速能否支撑估值

    # 维度3：price in 估算
    implied_growth_years: float
    # 当前估值隐含了多少年增长

    # 综合判断
    assessment: str  # undervalued / fair / expensive / extremely_expensive
    # "极度偏贵，PEG 0.83但分位94%，增速能支撑但历史位置太高"

    # 仓位建议
    suggested_max_weight: float
    # 极度偏贵 → 5%
    # 偏贵 → 8%
    # 合理 → 12%
    # 偏低 → 15%
```

### 4.5 组合构建模型

```python
class CognitionPortfolio:
    """认知匹配的组合方案"""

    selected_funds: list[PortfolioPosition]
    # 每只基金：代码 + 名称 + 权重 + 匹配度 + 估值 + 上限

    defense_position: PortfolioPosition | None
    # 防守仓位：红利低波基金，自动匹配

    overlap_analysis: dict[str, OverlapResult]
    # 基金间持仓重叠度

    correlation_matrix: dict[str, dict[str, float]]
    # 基金间NAV相关性

    risk_metrics: PortfolioRisk
    # 组合加权PE、波动率、最大回撤

    cash_pct: float
    # 现金比例
```

## 六、前端交互设计

### 6.1 核心理念

**不是"选基金"的界面，是"表达认知"的界面。**

投资人打开平台，第一眼看到的不是基金列表，而是：

```
┌──────────────────────────────────────────────────┐
│                                                  │
│                                                  │
│              你相信什么？                         │
│                                                  │
│     ┌──────────────────────────────────┐        │
│     │                                  │        │
│     │  输入你的投资认知...              │        │
│     │  或从常见认知中选择               │        │
│     │                                  │        │
│     └──────────────────────────────────┘        │
│                                                  │
│     [AI生产力变革]  [创新药错杀]  [龙头集中]     │
│     [消费升级]  [新能源]  [国产替代]             │

│     信心：○ 低  ● 中  ○ 高                      │
│     周期：○ 短期  ○ 中期  ● 长期                 │
│     风险：○ 保守  ● 适中  ○ 激进                 │
│                                                  │
│              [ 开始验证我的认知 ]                 │
│                                                  │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 6.2 四步引导流程

```
步骤1：认知输入
  → 选择主题 or 自定义输入
  → 展开产业链环节（可选择关注哪个环节）
  → 设置估值容忍度

步骤2：认知验证
  → 显示支持证据（绿色）
  → 显示反面证据（橙色）
  → 显示估值定价程度
  → 给出判断结论
  → "继续查看匹配基金" or "调整认知参数"

步骤3：基金穿透
  → 产业链分布图（哪个环节配了多少）
  → TOP基金列表（匹配度+估值分位+PEG+趋势）
  → 点击基金展开持仓穿透（底层股票+因子值）
  → 持仓变化趋势图

步骤4：组合方案
  → 配置清单（基金+权重+理由）
  → 持仓重叠度矩阵
  → 相关性矩阵
  → 组合风险指标
  → 防守仓位建议
  → "保存组合" or "导出报告"
```

### 6.3 与现有前端的关系

| 现有页面 | 定位 | 新增页面 |
|---|---|---|
| 基金诊断报告 | 单基金深度分析（保留） | 认知输入页（新增，核心入口） |
| 竞品横评 | 同类基金对比（保留） | 认知验证页（新增） |
| 组合工作台 | FOF筛选（保留） | 基金穿透页（新增） |
| 展示池 | 产品池管理（保留） | 组合方案页（新增） |

认知引擎的页面成为平台的**核心入口**，现有页面作为"深度工具"保留。

## 七、与标签引擎的整合

### 7.1 整合原则

```
认知引擎 = 上层（认知 → 组合）
标签引擎 = 下层（数据 → 证据）

认知引擎调用标签引擎的能力，但不依赖标签引擎的结论。
标签引擎提供"事实"，认知引擎提供"判断"。
```

### 7.2 整合点

```
认知引擎环节          调用标签引擎的什么
─────────────────────────────────────
产业链穿透      ←→    基金持仓数据 + 行业映射
估值门禁        ←→    股票因子（PE/PB/ROE/分位）
认知验证        ←→    收益风险标签 + 基准解析
持仓趋势        ←→    多期持仓数据
组合风险        ←→    波动率/回撤/Beta标签
基金相关性      ←→    NAV日收益数据
```

### 7.3 数据流

```
投资人输入认知
    ↓
认知引擎：结构化认知
    ↓
认知引擎：调用标签引擎获取持仓+因子数据
    ↓
认知引擎：产业链穿透 + 估值计算 + 趋势分析
    ↓
认知引擎：组合构建 + 风险分析
    ↓
输出：认知匹配的配置方案
```

标签引擎不需要改动，认知引擎直接读取其数据库。

## 九、核心设计原则

1. **认知是起点**：从"我相信什么"出发，不从"有什么数据"出发
2. **平台是镜子**：不告诉投资人"该买什么"，而是帮投资人看清"自己的认知意味着什么"
3. **穿透到底层**：基金是壳子，核心是底层资产
4. **估值是硬约束**：热爱 ≠ 买入
5. **提供证据不替代判断**：信息矛盾时靠"相信"
6. **组合思维**：不是选最好的基金，是选最匹配认知且风险可控的组合
7. **标签引擎是底座**：它提供事实，认知引擎提供判断
8. **动态跟踪**：认知在变、估值在变、持仓在变

- 看清自己买的基金到底装了什么
- 知道贵不贵、该配多少
