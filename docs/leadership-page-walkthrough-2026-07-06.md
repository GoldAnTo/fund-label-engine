# 基金标签研究台页面讲解版

> 适用场景：给领导或非技术同事口头介绍当前项目页面。
> 讲解目标：让对方明白系统已经做了什么、每个页面解决什么问题、结果为什么可信、哪些地方还需要继续补。
> 页面依据：当前前端路由和页面实现，包括展示池、风格筛选、竞品横评、组合工作台、复核队列、批次管理、批次详情、批次对比、单基金报告。

---

## 1. 开场怎么讲

可以先用这一段开场：

> 这个系统不是一个简单的基金列表页面，也不是推荐买卖的工具。它现在更像一个基金研究生产线：先从 fundData 取基金基础数据、净值、持仓、经理、费率等材料，再叠加股票因子和行业映射，批量生成标签、证据、分类、分组和展示状态。前端页面的作用，就是把这条生产线变成业务人员能看、能筛、能复核、能横向比较、能进入组合讨论的工作台。

再补一句边界：

> 所有页面展示的是标签和证据，不直接给投资建议。系统强调的是可解释、可追溯、可复核，而不是替研究员做最终判断。

---

## 2. 推荐演示顺序

建议按这条线讲，最顺：

```text
展示池与风格总览
  -> 风格筛选
  -> 单基金报告
  -> 竞品横评
  -> 组合工作台
  -> 复核队列
  -> 批次管理 / 批次详情 / 批次对比
```

这条线对应一个完整业务闭环：

```text
先看整体有多少基金可展示
  -> 再按风格找基金
  -> 再打开单只基金看证据
  -> 再把几只基金放一起横评
  -> 再进入组合草案
  -> 再把有问题的样本交给人工复核
  -> 最后用批次页面证明整个过程可重复、可审计
```

## 3. 页面一：展示池与风格总览

对应入口：

- 侧边栏「风格总览」
- 侧边栏「展示池」
- 当前实现里 `/explorer` 和 `/ready-pool` 都复用展示池总览能力

### 这一页解决什么问题

这一页回答的是：

```text
这一批基金里，哪些结果能正式展示？
哪些暂时不能展示？
不能展示是因为缺基准、缺收益窗口，还是需要确认映射？
```

### 可以怎么讲

> 这里是项目的第一屏，先不看单只基金，而是看整批基金的展示资格。系统会把基金分成可展示和暂不可展示两类。可展示不是人工拍脑袋，而是需要同时满足数据、收益窗口和基准质量这些门禁。

> 这个页面的价值是把“能不能对外展示”这件事产品化了。过去我们可能只知道某只基金没算出来，现在页面会告诉我们具体卡在哪里，比如缺基准收益源、收益窗口不足、基准未配置或组件还没解析。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| 当前批次 | 说明每次结果都属于某一次跑批，不是静态手工表 |
| 基金池数量 | 说明当前分析范围 |
| 可展示 / 不可展示统计 | 说明系统有正式展示门禁 |
| 阻塞原因分布 | 说明不能展示不是失败，而是清楚标出了缺口 |
| 风格分布 / Top 基金 | 说明系统已经能把标签结果转成研究视角 |
| 单基金入口 | 说明每个总览数字都能下钻到证据 |

### 领导可能问

**问：不可展示是不是代表基金不好？**

答：

> 不是。不可展示代表当前证据链不完整，比如基准收益源不够或收益窗口不足。它不是基金好坏判断，而是展示合规性和数据充分性的判断。

**问：为什么不直接用沪深 300 代替缺失基准？**

答：

> 因为不同基金合同里的业绩比较基准不同。如果随便用沪深 300，Alpha、Beta、超额收益都可能错。系统现在的原则是宁可标记缺数据，也不用错误基准做代理。

## 4. 页面二：风格筛选

对应入口：

- 侧边栏「风格筛选」
- 路由 `/search`

### 这一页解决什么问题

这一页回答的是：

```text
如果我想找某一类风格基金，系统能不能筛出来？
```

比如：

- 低估值基金有哪些？
- 质量成长基金有哪些？
- 高估值基金有哪些？
- 大盘、中盘、小盘风格分别有哪些？
- 科技、消费、医药等主题基金分别有哪些？

### 可以怎么讲

> 展示池解决的是“能不能展示”，风格筛选解决的是“怎么找基金”。这里把底层标签变成了业务可用的筛选条件。研究员不用去数据库查 label_code，而是直接按风格维度点选。

> 筛出来以后，页面不会只给一个基金代码，而是会带上它命中的风格标签、复核状态和单基金报告入口。这样就能从“风格池”继续下钻到“单只基金为什么命中这个风格”。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| 风格快捷筛选 | 把技术标签翻译成研究语言 |
| 基金代码搜索 | 支持单只基金定位 |
| 筛选结果 | 每只基金带风格标签和复核状态 |
| 查看报告 | 所有筛选结果都能回到证据链 |

### 讲解提醒

这里不要把风格标签说成永久结论。更稳妥的说法是：

> 当前风格主要基于最新持仓和股票因子，是一个可解释的当前快照。后续如果积累多期持仓，就可以进一步讲风格稳定性和风格漂移。

## 5. 页面三：单基金报告

对应入口：

- 从展示池、风格筛选、竞品横评、批次详情进入
- 路由 `/runs/:runId/funds/:fundCode`

### 这一页解决什么问题

这一页回答的是：

```text
这只基金为什么被打这些标签？
哪些指标支持？
哪些标签没触发？
哪些数据缺失？
是否能进入展示池？
```

### 可以怎么讲

> 单基金报告是整个系统的证据核心。总览页和筛选页只告诉我们“它是什么”，单基金报告告诉我们“为什么”。这里会展示这只基金的展示资格、年化收益、基准情况、风格标签、相对基准标签、同类分位、因子暴露、标签详情、未触发原因和复核记录。

> 这个页面最重要的不是标签数量，而是每个标签后面都有证据。比如风格标签可以下钻到持仓股票，看哪些股票贡献了这个风格；相对基准标签可以看到基准组件是否解析、有没有收益源；未触发标签也会说明是没达到阈值，还是数据不够不能算。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| 报告头 | 展示资格、年化收益、基准收益、标签数量 |
| 决策卡片 | GO / WATCH，把展示状态讲清楚 |
| 风格标签 | 可以下钻到持仓贡献明细 |
| 相对基准 | 说明 Alpha、Beta、超额收益依赖基准质量 |
| 基准解析 | 展示基准组件、权重、解析状态、收益源 |
| 同类分位排名 | 不只看绝对值，还看同类位置 |
| 因子暴露 | 解释风格标签的底层数据 |
| 标签详情 | 每个标签有证据 message |
| 未触发标签原因 | 说明没打标签的原因也被记录 |
| 导出按钮 | 支持 CSV / XLSX 交付 |
| 复核入口 | 研究员可以保留人工判断 |

### 可以重点讲的一句话

> 这页证明系统不是黑盒。它不只是说“这只基金是质量成长”，还会告诉我们哪些持仓股票、哪些因子、哪些阈值共同支撑这个判断。

## 6. 页面四：竞品横评

对应入口：

- 侧边栏「竞品横评」
- 路由 `/compare`

### 这一页解决什么问题

这一页回答的是：

```text
几只看起来相似的基金，具体差别在哪里？
它们是不是买了类似的股票？
相关性高不高？
放在一起能不能分散风险？
```

### 可以怎么讲

> 竞品横评是给投研横向比较用的。单基金报告是纵向体检，竞品横评是横向对照。页面支持选择 2 到 6 只基金，比较核心指标、风格雷达图、风格标签矩阵、持仓重叠度、收益相关性和组合风险预估。

> 这解决了一个很实际的问题：两只基金都叫成长或都属于科技主题，但它们可能收益风险不同、持仓重叠不同、相关性也不同。这个页面把这些差异放到同一张桌子上看。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| 选基区 | 支持 2 到 6 只基金横评 |
| 核心指标对比 | 收益、回撤、夏普等指标并排，标出最优 |
| 分位排名 | 不只比数值，也看全市场位置 |
| 风格雷达图 | 用可视化展示风格差异 |
| 风格标签矩阵 | 看哪些标签共同命中、哪些不同 |
| 相关性矩阵 | 判断基金之间替代性或分散性 |
| 高相关基金对 | 提醒“可能二选一”的基金 |
| 组合风险预估 | 调整权重，看组合波动和分散效果 |

### 可以重点讲的一句话

> 这一页把“基金标签”往前推进了一步：不只是给基金画像，而是帮助研究员比较同类基金，判断它们是互补还是重复。

## 7. 页面五：组合工作台

对应入口：

- 侧边栏「组合工作台」
- 路由 `/portfolio`

### 这一页解决什么问题

这一页回答的是：

```text
标签结果怎么进入组合讨论？
哪些基金适合作为核心？
哪些适合作为卫星？
哪些只是指数工具？
哪些应该排除或等待复核？
```

### 可以怎么讲

> 组合工作台是把标签结果转成业务动作的页面。前面的页面解决“看清基金”，这个页面解决“怎么把基金放进组合讨论”。系统会根据标签、风险、数据状态、基准状态和人工复核结果，把基金映射到核心、卫星、指数工具、排除等角色。

> 页面有两个口径：研究草案和已验收组合。研究草案更宽，用来讨论；已验收组合更严，只保留人工签核过的基金。这一点很重要，因为它把自动计算和人工决策分开了。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| Executive overview | 一屏说明项目不是标签小工具，而是研究基础设施 |
| 已处理基金 / 可展示池 / 组合草案 / 人工签核 | 四个指标说明业务闭环 |
| 生产线说明 | 数据接入 -> 标签计算 -> 证据落库 -> 展示门禁 -> 人工签核 -> 组合草案 |
| 基金地图 | 每只基金的角色、标签、风险、阻塞原因 |
| 草案桶 | 核心、卫星、指数工具等组合结构 |
| 单基金侧栏 | 选中基金后能看标签、证据、阻塞原因 |
| 人工角色复核 | 研究员可以 accept / reject / needs_more_data |
| 建议采纳 | 支持根据系统建议批量生成复核动作 |

### 讲解时要强调的边界

> 组合工作台不是直接给最终组合，也不是自动下单。它输出的是可讨论、可复核的组合草案。真正进入组合仍然需要研究员签核和业务决策。

## 8. 页面六：复核队列

对应入口：

- 侧边栏「复核队列」
- 路由 `/review-queue`

### 这一页解决什么问题

这一页回答的是：

```text
系统自动计算后，哪些事情还需要人处理？
优先级是什么？
应该去哪里处理？
```

### 可以怎么讲

> 复核队列是自动化和人工判断之间的交接区。系统不会把所有不确定样本硬算成结论，而是把基准缺口、人工复核、观察信号、待校准信号汇总成任务队列。

> 这页的价值是管理“未完成事项”。领导看到这里能知道，系统不是只报喜，也会把风险、缺口和待处理项摆出来。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| 全部任务 | 当前批次总共有多少待处理 |
| 基准缺口 | 哪些问题来自 benchmark 数据 |
| 人工复核 | 哪些样本需要研究员判断 |
| 观察 / 校准 | 哪些标签或样本需要后续规则校准 |
| 优先级 | 任务有高、中、低区分 |
| 建议动作 | 系统给出下一步处理建议 |
| 导出 CSV / XLSX | 复核任务可以交付给人工处理 |

### 可以重点讲的一句话

> 复核队列让系统从“一次计算”变成“持续运营”：每次跑完都知道哪些结果能用、哪些问题要补、补什么。

这一页回答的是：

```text
标签结果是不是一次性的？
能不能重复跑？
```

### 页面上重点指给领导看的地方

## 10. 页面八：批次详情与审计

对应入口：

- 从批次管理点「查看」
- 路由 `/runs/:runId`

### 这一页解决什么问题

这一页回答的是：

```text
某一次跑批到底处理了多少基金？
有没有失败？
命中了哪些标签？
规则快照是什么？
和上一次相比有没有风险变化？
```

### 可以怎么讲

> 批次详情是审计页。它记录一次跑批的完整快照，包括处理结果、失败原因、标签分布、风格分布、标签变化、规则快照和导出能力。

> 这个页面对管理很重要，因为它让结果可以追溯。后续如果有人问“这个标签是什么时候算出来的、当时用的规则是什么、有没有失败样本”，都可以从这里查。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| 决策摘要 | 本批处理基金数、失败数、成功率 |
| 核心指标卡 | 已处理、失败、数据不足、需人工复核、收益窗口不足 |
| 标签命中分布 | 看本批主要标签分布 |
| 风格分布 | 看风格标签命中情况 |
| 标签变化 | 看相对上次新增、消失、状态变更 |
| 导出按钮 | 批次结果可以导出交付 |

## 11. 页面九：批次对比

对应入口：

- 侧边栏「批次对比」
- 路由 `/diff`

### 这一页解决什么问题

这一页回答的是：

```text
两次跑批之间，标签结果发生了什么变化？
是新增了标签，消失了标签，还是基金范围变了？
```

### 可以怎么讲

> 批次对比用于做标签变动复盘。它把旧批次和新批次放在一起比较，只统计两批共有基金的标签变化，同时把单边出现的基金单独列出来，避免把基金范围变化误判成标签变化。

> 这个页面对后续运营很关键，因为真实系统不是只跑一次，而是每月、每季度持续跑。批次对比可以告诉研究员：哪些标签变了，为什么要复盘。

### 页面上重点指给领导看的地方

| 区域 | 讲解重点 |
|---|---|
| base / target | 旧批次和新批次 |
| 总览 | 共同基金数、新增标签、消失标签、有变动基金数 |
| 按标签 | 看哪个标签变动最大 |
| 按基金 | 看哪只基金标签变化最多 |
| 单边基金 | 避免误把范围变化当成标签变化 |

## 12. 整体页面闭环怎么总结

可以用这一段收束：

> 现在前端页面已经覆盖了基金研究从结果展示到业务动作的主要链路。展示池告诉我们哪些基金能正式展示；风格筛选让研究员按标签找基金；单基金报告解释每个标签的证据；竞品横评支持基金之间比较；组合工作台把标签转成核心、卫星、指数工具等组合角色；复核队列承接自动化不能直接下结论的样本；批次管理和批次对比保证整个过程可以重复、可以审计、可以追踪变化。

再用一句更短的：

> 所以这个项目现在不是“页面展示了一些标签”，而是已经形成了一个基金标签生产、解释、复核、比较和组合应用的闭环。

---

## 13. 讲解时建议强调的三个价值

### 价值一：把口径统一

> 所有标签都来自同一套规则配置和同一套跑批流程，避免不同人对“成长”“价值”“高波动”“可展示”理解不一致。

### 价值二：把证据留下来

> 页面不是只展示结论，还展示指标值、阈值、基准组件、因子暴露、持仓贡献、未触发原因和复核记录。

### 价值三：把缺口暴露出来

> 数据不足、基准缺失、收益窗口不足、需要人工复核的样本不会被隐藏，而是进入展示池阻塞原因和复核队列。

## 14. 讲解时不要过度承诺的地方

| 不要这样说 | 更稳妥的说法 |
|---|---|
| 系统能推荐基金 | 系统能提供标签、证据和组合草案，最终决策仍由研究员和业务流程完成 |
| 不可展示的基金不好 | 不可展示代表证据链暂不完整，不代表基金质量差 |
| 风格标签是永久风格 | 当前风格主要基于最新持仓和当前因子快照，后续需要多期数据验证稳定性 |
| Alpha/Beta 全部可用 | 只有基准收益源和收益窗口通过门禁的基金才展示相对基准标签 |
| 组合工作台能直接生成最终组合 | 现在是研究草案和签核工具，不是自动投资决策 |

## 15. 5 分钟讲解版

如果时间很短，可以这样讲：

> 第一屏展示池，先看这批基金哪些可以正式展示，哪些因为基准、收益窗口等原因暂时不能展示。

> 第二步风格筛选，按低估值、质量成长、红利、大盘、小盘、行业主题等标签找基金。

> 第三步打开单基金报告，看这只基金为什么命中这些标签：指标、阈值、基准组件、因子暴露、持仓贡献和未触发原因都能看到。

> 第四步竞品横评，把 2 到 6 只基金放一起比收益、风险、风格、持仓重叠和相关性，判断它们是互补还是重复。

> 第五步组合工作台，把标签结果映射到核心、卫星、指数工具、排除等组合角色，并保留研究员人工签核。

> 最后是复核队列和批次审计。系统会把不能自动下结论的样本放进复核队列，每次跑批也都有 run_id、规则版本、标签变化和导出结果，保证过程可追溯。

## 16. 15 分钟讲解版

如果有 15 分钟，可以按这个节奏：

1. 用 1 分钟讲项目定位：基金体检和标签报告系统，不是推荐交易系统。
2. 用 2 分钟讲展示池：整体基金池、可展示数量、阻塞原因、为什么不能乱用代理基准。
3. 用 2 分钟讲风格筛选：如何按风格找基金，如何进入单基金报告。
4. 用 4 分钟讲单基金报告：展示资格、标签证据、基准解析、因子暴露、未触发原因、导出和复核。
5. 用 2 分钟讲竞品横评：指标对比、风格雷达、标签矩阵、相关性、组合风险。
6. 用 2 分钟讲组合工作台：标签如何转成核心/卫星/指数工具，研究草案和已验收组合的区别。
7. 用 1 分钟讲复核队列：自动化边界和人工任务。
8. 用 1 分钟讲批次审计：可重复、可导出、可对比、可追溯。

## 17. 最后一页总结话术

> 这个项目现在已经完成了从数据底座到前端工作台的第一版闭环。后端负责把基金数据批量加工成标签、证据、分类、分组和展示状态；前端负责把这些结果变成研究员能使用的页面，包括展示池、风格筛选、单基金报告、竞品横评、组合工作台、复核队列和批次审计。

> 它目前最核心的价值是三点：第一，统一基金标签口径；第二，保留每个标签背后的证据链；第三，把不能直接展示或需要人工判断的问题显式暴露出来。后续继续补基准数据源、积累多期持仓和标签时序后，这套系统就可以从单次体检继续升级成持续跟踪和组合研究工具。
