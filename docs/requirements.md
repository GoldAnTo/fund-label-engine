# 需求文档

## 目标

建设一个基金标签计算引擎，把基金数据转化为可解释、可追溯、可复核的标签结果。系统服务于证券资管公司的产品池准入、观察池管理、基金画像、人工复核和后续 FOF 研究。

## 第一版用户

- 投研人员：查看基金画像、收益风险、持仓结构、标签证据。
- 产品池审核人员：判断基金是否进入观察池、是否需要人工复核。
- 数据/系统开发人员：维护数据契约、标签定义、计算批次和结果审计。

## 第一版输入

- 基金基础信息：代码、名称、类型、成立日期、规模、基金公司。
- 净值/收益序列：日频或周频净值、阶段收益。
- 持仓数据：股票持仓、债券持仓、行业配置。
- 基金经理：经理名称、任期、变更情况。
- 费率数据：管理费、托管费、销售服务费。
- 可选股票因子：PB、ROE、股息率、营收增长、利润增长、市值、行业、估值分位。

## 第一版输出

- 数据覆盖率和缺失项。
- 基金特征值，例如最大回撤、波动、夏普、持仓集中度、行业集中度。
- 标签结果，例如 `data_sufficient`、`holding_concentration_high`、`manager_tenure_long`。
- 标签证据，例如指标值、阈值、数据来源、解释文本。
- 审核建议，例如 `observe`、`manual_review`、`not_ready`。

## 必须满足的原则

1. 每个标签必须能解释。
2. 每个标签必须保存证据。
3. 数据不足时不强行输出正式标签。
4. 高级风格标签必须依赖股票因子或股票标签。
5. 规则需要可版本化，方便未来回溯历史计算。
6. 人工复核和自动计算需要分开保存。

## MVP 验收

- 能对单只基金执行标签计算。
- 能批量计算一组基金。
- 能输出数据覆盖率、标签、证据和复核建议。
- 缺少股票因子时，不输出深度价值、质量成长、红利稳健等正式风格标签。
- 测试覆盖覆盖率检查、基础标签、证据生成和高级标签边界。
