## 一、当前演示为什么太简单

### 1.1 演示做了什么

上一版演示：输入"我相信AI" → 按 sector_group='tech' + 20只关键个股名称匹配 → 输出匹配度排名 + 加权PE。

### 1.2 头部机构看这有什么问题

| 维度 | 演示做的 | 机构需要的 | 差距 |
|---|---|---|---|
| 认知输入 | 主题关键词"AI" | 完整投资逻辑：AI基础设施先于应用层变现，因为收入确认周期不同 | 认知没有结构化，无法验证 |
| 资产匹配 | sector_group='tech' | 产业链穿透：芯片设计→晶圆代工→封装→服务器→光模块 | 一个"tech"标签把半导体和软件混在一起 |
| 估值判断 | PE>50 = 偏贵 | 历史分位 + 横截面对比 + price in估算 | 只有绝对值，不知道在历史什么位置 |
| 认知验证 | 无 | 正反证据框架：基本面支撑 vs 估值风险 vs 市场情绪 | 完全缺失 |
| 组合构建 | 选TOP3各配8% | 相关性分析 + 持仓重叠度 + 风险预算 | 可能选了3只买同一批股票的基金 |
| 持续跟踪 | 最新一期快照 | 持仓变化趋势 + 估值变化趋势 | 不知道基金在加仓还是减仓 |

### 1.3 根本问题

**演示只做了"匹配"，没做"投研"。** 头部机构的投研不是"主题→基金"的简单映射，而是一套从假设到验证到组合到跟踪的完整方法论。认知引擎要做的不是替代这套方法论，而是把它数字化、结构化、可验证。

## 二、头部机构投研流程拆解

### 2.1 从认知到组合的五个环节

```
环节1：认知形成
  研究员形成投资判断："我相信X因为Y"
  - 不是模糊的感觉，而是有逻辑链的假设
  - 例：AI基础设施会先于应用层变现，因为服务器/芯片的收入确认
    先于大模型公司的商业化

环节2：认知验证
  用数据验证这个假设成不成立
  - 基本面：行业增速、企业盈利、订单数据
  - 估值：当前定价是否已反映这个认知
  - 市场情绪：资金流向、拥挤度
  - 反面证据：有没有数据不支持这个认知

环节3：资产穿透
  把认知映射到具体的底层资产
  - 不是"tech行业"，而是产业链环节
  - AI算力链：设计(寒武纪)→代工(台积电)→封装(日月光)→
    模块(中际旭创)→服务器(工业富联)→数据中心
  - 每个环节的供需格局、竞争壁垒、盈利周期不同

环节4：组合构建
  把资产组合成可执行的投资方案
  - 认知匹配度 → 选哪些基金
  - 估值约束 → 每个配多少
  - 相关性约束 → 避免集中
  - 风险预算 → 组合整体风险可控
  - 对冲仓位 → 自动配置防守

环节5：持续跟踪
  认知不是一成不变的
  - 基本面变化 → 认知是否被验证/证伪
  - 估值变化 → 是否已过度定价
  - 持仓变化 → 基金经理是否在调整方向
  - 预警机制 → 什么时候该调仓
```

### 2.2 现有标签引擎在哪个位置

```
认知形成 → 认知验证 → 资产穿透 → 组合构建 → 持续跟踪
                        ↑
                   标签引擎在这里
                   它提供"证据"
```

标签引擎不是产品出口，而是认知验证和资产穿透的证据底座。它的价值在于：
- 持仓数据 → 用于资产穿透
- 股票因子 → 用于估值检查
- 行业映射 → 用于产业链匹配
- 收益风险标签 → 用于组合风险约束
- 基准解析 → 用于相对表现验证

## 三、深度认知引擎设计

### 3.1 认知结构化：从模糊感觉到可验证假设

#### 当前问题

演示中认知输入是 `"AI"` 一个关键词。但真实的投资认知是一个完整的逻辑链。

#### 机构级设计

一个认知应该包含五个要素：

```yaml
cognition:
  # 1. 核心判断：你相信什么
  belief: "AI基础设施会先于应用层变现"

  # 2. 逻辑链：为什么相信
  logic_chain:
    - "AI服务器/芯片的收入确认先于大模型商业化"
    - "基础设施公司（台积电/美光）已确认大量收入"
    - "应用层公司（OpenAI/MiniMax）仍在亏损"
    - "因此基础设施环节的确定性更高"

  # 3. 产业链环节：具体投哪个环节
  chain_focus:
    primary: "AI算力基础设施"       # 主要投这个环节
    secondary: "AI芯片设计"         # 次要配置
    exclude: ["AI应用层"]           # 暂时排除

  # 4. 估值容忍度：能接受多贵
  valuation_tolerance:
    max_pe: 80                     # PE超过80不考虑
    max_valuation_percentile: 85   # 历史分位超过85%不追
    require_margin_of_safety: false # 不要求安全边际（成长股）

  # 5. 信心与时间
  conviction: "high"               # 信心强度
  time_horizon: "2-3年"            # 投资周期
  risk_tolerance: "moderate"       # 风险偏好
```

#### 为什么这样设计

- **逻辑链**让认知可验证：每一条都可以用数据检验
- **产业链环节**让匹配更精准：不是泛泛的"科技"，而是具体的算力链路
- **估值容忍度**是个性化的：不同投资者对估值的接受度不同
- **信心强度**影响仓位：高信心多配，低信心少配或不配

### 3.2 产业链穿透：从主题到资产到基金

#### 当前问题

演示用 `sector_group='tech'` 匹配，把半导体、软件、通信设备全混在一起。但AI算力链和AI软件的投资逻辑完全不同。

#### 机构级设计

建立产业链图谱，每个认知主题拆解到具体环节：

```yaml
theme: "AI"
chain_links:
  # 环节1：芯片设计
  chip_design:
    stocks: ["寒武纪", "海光信息", "澜起科技"]
    sectors: ["半导体-芯片设计"]
    characteristics:
      - "高研发投入"
      - "国产替代逻辑"
      - "估值通常较高（PE 50-200）"

  # 环节2：晶圆代工
  foundry:
    stocks: ["台积电", "中芯国际"]
    sectors: ["半导体-晶圆代工"]
    characteristics:
      - "资本密集型"
      - "产能利用率驱动"
      - "估值相对合理（PE 15-30）"

  # 环节3：封装测试
  packaging:
    stocks: ["长电科技", "通富微电", "华天科技"]
    sectors: ["半导体-封装测试"]
    characteristics:
      - "先进封装受益于AI芯片"
      - "估值较低（PE 15-25）"

  # 环节4：光模块/连接
  optical_module:
    stocks: ["中际旭创", "天孚通信", "新易盛"]
    sectors: ["通信设备-光模块"]
    characteristics:
      - "AI数据中心直接受益"
      - "业绩弹性大"
      - "估值偏高（PE 50-150）"

  # 环节5：服务器/算力
  server:
    stocks: ["工业富联", "浪潮信息", "紫光股份"]
    sectors: ["计算机设备-服务器"]
    characteristics:
      - "AI服务器渗透率提升"
      - "估值中等（PE 20-40）"
```

#### 匹配逻辑升级

不是简单的"行业占比"，而是：

```
基金A持仓分析：
  光模块环节占比：15.2%（中际旭创9.8% + 天孚通信5.4%）
  芯片设计环节占比：8.5%
  服务器环节占比：6.3%
  AI算力链总匹配：30.0%

  vs 演示中的"tech行业占比98.3%"
  → 产业链匹配更精准，知道基金A主要投的是光模块环节
```

#### 用现有数据验证

现有 `stock_industry_map` 有 `industry_name` 字段（如"电子设备-电子元件"、"通信设备-光通信"），可以用来做更细的产业链匹配，而不只是靠 `sector_group` 的 8 大类。

### 3.3 估值多维判断：不只是PE绝对值

#### 当前问题

演示用 `PE>50 = 偏贵` 的绝对值判断。但不同行业的合理PE范围不同：银行PE 10正常，半导体PE 50可能合理。

#### 机构级设计

估值判断应该是三维的：

```
维度1：历史分位
  当前PE在过去N年的什么位置？
  - 数据：stock_factor_values 中的 valuation_percentile（已有！5485只股票）
  - 输出：当前估值分位85% → 比过去85%的时间都贵

维度2：横截面对比
  跟同行业其他公司比贵不贵？
  - 数据：同行业股票的PE中位数
  - 输出：该基金持仓PE 107，同行业PE中位数45 → 溢价2.4倍

维度3：price in 估算
  当前估值隐含了多少年增长？
  - 逻辑：PE 100 + 利润增速50% → PEG 2.0 → 隐含约2年高增长
  - 如果增速无法持续，估值就会回归
  - 输出：当前估值已price in约2年50%增速
```

#### 用现有数据验证

```sql
-- 已有 valuation_percentile 因子，可以直接用
SELECT stock_code, factor_value AS valuation_pct
FROM stock_factor_values
WHERE factor_code = 'valuation_percentile'
-- 输出：300308(中际旭创) valuation_pct = 0.986 → 历史最高位附近
```

```sql
-- 已有 profit_growth 因子，可以算PEG
SELECT stock_code,
  pe.factor_value AS pe,
  pg.factor_value AS profit_growth,
  pe.factor_value / (pg.factor_value * 100) AS peg
FROM stock_factor_values pe
JOIN stock_factor_values pg ON pe.stock_code = pg.stock_code
WHERE pe.factor_code = 'pe' AND pg.factor_code = 'profit_growth'
-- 输出：300308 PE=97.7, 增速=90%, PEG=1.08 → 增速能支撑估值
```

### 3.4 认知验证：正反证据框架

#### 当前问题

演示完全缺失认知验证。只做了匹配，没有检验认知本身。

#### 机构级设计

对每个认知，系统提供四个维度的验证证据：

```
认知：AI基础设施会先于应用层变现

┌── 支持证据 ──────────────────────────────────────┐
│ ✅ 基本面：AI基础设施公司利润增速 90%+             │
│ ✅ 收入确认：台积电/美光已确认大量AI相关收入       │
│ ✅ 产能扩张：全球晶圆代工产能利用率 95%+           │
└──────────────────────────────────────────────────┘

┌── 反面证据 ──────────────────────────────────────┐
│ ⚠️ 估值风险：光模块板块PE分位 98%，接近历史最高    │
│ ⚠️ 周期风险：半导体周期通常2-3年，当前可能在顶部   │
│ ⚠️ 集中度：AI算力链持仓高度集中在少数股票          │
└──────────────────────────────────────────────────┘

┌── 估值定价 ──────────────────────────────────────┐
│ 当前定价程度：已price in约2年高增长                 │
│ 安全边际：低（估值分位85%+）                       │
│ 结论：认知方向正确，但市场已充分定价，赔率不佳      │
└──────────────────────────────────────────────────┘

┌── 市场情绪 ──────────────────────────────────────┐
│ 板块涨幅：近6个月 +35%（超额收益明显）             │
│ 拥挤度：高（多只基金重仓相同股票）                 │
│ 结论：短期可能面临回调压力                         │
└──────────────────────────────────────────────────┘

最终判断：认知有效但已过度定价
建议：方向正确，但需等估值回落后加仓，或降低仓位
```

#### 验证框架的数据需求

| 验证维度 | 需要什么数据 | 现有数据 | 需要补充 |
|---|---|---|---|
| 基本面 | 利润增速、营收增速 | ✅ profit_growth, revenue_growth | — |
| 估值定价 | PE、PB、估值分位 | ✅ pe, pb, valuation_percentile | — |
| 持仓集中度 | 基金持仓重叠度 | ✅ stock_holdings | — |
| 板块涨幅 | 行业指数收益 | ✅ benchmark_returns | 需补充行业指数 |
| 市场情绪 | 资金流向 | ❌ | 需接入新数据源 |

### 3.5 组合风险管理：不只是选基金

#### 当前问题

演示只选了TOP基金，没有分析：选的3只基金是不是买同一批股票？组合整体风险多大？

#### 机构级设计

组合构建必须考虑四个维度：

```
维度1：持仓重叠度
  基金A TOP10持仓 vs 基金B TOP10持仓
  → 重叠股票合计权重 42%
  → 结论：这两只基金本质上是"同一个基金"，只需选一只

维度2：相关性分析
  基金A和基金B的日收益相关系数 0.92
  → 高度相关，分散效果差
  → 需要找相关性低的基金来分散

维度3：组合风险预算
  组合加权波动率 = Σ(wi × σi × ρij × wj)
  组合最大回撤 = 历史模拟
  组合VaR = 95%置信度下的最大损失
  → 设定风险上限：组合波动不超过20%，回撤不超过15%

维度4：风格暴露约束
  组合级行业分布：
  - 科技 35% → 超配（基准15%）
  - 医药 10% → 标配
  - 金融 15% → 低配（防守）
  → 行业偏离超过±10%时预警
```

#### 用现有数据验证

```sql
-- 持仓重叠度：两只基金的共同持仓
SELECT a.stock_code, a.stock_name,
  a.net_value_ratio AS fund_a_weight,
  b.net_value_ratio AS fund_b_weight,
  a.net_value_ratio + b.net_value_ratio AS combined
FROM stock_holdings a
JOIN stock_holdings b ON a.stock_code = b.stock_code
WHERE a.fund_code = '000522' AND b.fund_code = '000063'
  AND a.report_period = '2025-12-31'
  AND b.report_period = '2025-12-31'
ORDER BY combined DESC

-- 相关性：用NAV日收益算
SELECT a.nav_date, a.daily_growth_rate AS fund_a, b.daily_growth_rate AS fund_b
FROM nav_history a
JOIN nav_history b ON a.nav_date = b.nav_date
WHERE a.fund_code = '000522' AND b.fund_code = '000063'
```

### 3.6 持仓变化趋势：不是静态快照

#### 当前问题

演示只看了最新一期持仓。但会议说"线性外推不成立"——如果基金在减仓AI，你还按最新持仓匹配就会误导。

#### 机构级设计

追踪持仓变化趋势：

```
基金000522 持仓变化趋势（近4个报告期）：

  AI算力链持仓权重变化：
  2024-12-31:  45.2%
  2025-03-31:  52.8%  ↑ +7.6%
  2025-06-30:  68.5%  ↑ +15.7%
  2025-12-31:  83.2%  ↑ +14.7%

  趋势判断：持续加仓AI算力链，方向坚定
  → 认知匹配度可信

  vs 另一只基金：
  2024-12-31:  65.3%
  2025-06-30:  48.2%  ↓ -17.1%
  2025-12-31:  32.5%  ↓ -15.7%

  趋势判断：持续减仓AI算力链
  → 虽然最新仍有32.5%匹配度，但方向在撤
  → 认知匹配度需打折
```

#### 用现有数据验证

```sql
-- 有多个报告期的持仓数据
SELECT report_period, COUNT(DISTINCT fund_code)
FROM stock_holdings
GROUP BY report_period
-- 输出：2024Q1到2025Q4都有数据，可以看变化趋势
```

## 四、完整链路：从认知到组合

### 4.1 端到端流程

```
步骤1：认知输入
  "我相信AI基础设施会先于应用层变现"
  + 逻辑链 + 产业链环节 + 估值容忍度 + 信心强度

步骤2：认知验证
  ✅ 基本面：基础设施公司利润增速90%+
  ⚠️ 估值：PE分位85%+，已price in 2年增长
  ⚠️ 拥挤：多只基金重仓相同股票
  → 判断：认知有效但已过度定价

步骤3：产业链穿透
  AI算力链拆解：
  - 光模块环节 → 中际旭创/天孚通信
  - 芯片设计 → 寒武纪/海光信息
  - 服务器 → 工业富联/浪潮信息
  → 扫描142只基金持仓，按环节计算匹配度

步骤4：基金筛选
  匹配度 + 估值约束 + 持仓趋势
  → 基金A：匹配度83%，PE 107，持续加仓 → 候选（但估值偏高）
  → 基金B：匹配度45%，PE 35，持续加仓 → 候选（估值合理）
  → 基金C：匹配度60%，PE 80，开始减仓 → 排除（趋势转向）

步骤5：组合构建
  基金A + 基金B 持仓重叠度 = 38% → 可接受
  基金A + 基金B 相关性 = 0.85 → 偏高，需分散
  → 加入防守仓位：红利低波基金（PE 10，股息率4.9%）
  → 组合风险预算：波动率 < 20%

步骤6：组合输出
  基金A：8%（认知匹配，但估值约束仓位）
  基金B：12%（认知匹配，估值合理，多配）
  红利防守：10%（对冲认知错误风险）
  现金：70%

  组合加权PE：32倍
  组合年化波动：15%
  持仓穿透：中际旭创5.2%、工业富联3.8%...
```

### 4.2 与现有标签引擎的整合点

| 认知引擎环节 | 调用标签引擎的什么能力 |
|---|---|
| 产业链穿透 | 基金持仓数据、行业映射 |
| 估值检查 | 股票因子（PE/PB/ROE/估值分位） |
| 基金筛选 | 收益风险标签、基准解析、Alpha/Beta |
| 组合风险约束 | 波动率标签、回撤标签、Beta |
| 持仓变化趋势 | 多期持仓数据 |
| 持仓重叠度 | 两只基金的持仓数据 |

## 五、现有数据能支撑什么

### 5.1 可以立即实现的

| 能力 | 数据来源 | 实现方式 |
|---|---|---|
| 产业链穿透（按行业细分） | stock_industry_map.industry_name | 用细分行业名替代粗分sector_group |
| 估值历史分位 | valuation_percentile因子 | 直接读取，已覆盖5485只股票 |

## 七、核心原则

1. **认知是起点，不是数据**：从"我相信什么"出发，不是从"有什么数据"出发
2. **穿透到底层资产**：基金是壳子，核心是它装了什么、为什么装这些
3. **估值是硬约束**：热爱不等于买入，估值太贵时必须提示风险
4. **提供证据不替代判断**：信息矛盾时靠"相信"，系统给证据不给结论
5. **组合思维不是单选**：不是选最好的基金，而是选最匹配认知且风险可控的组合
6. **动态跟踪不是静态快照**：持仓在变、估值在变、认知也在变
7. **标签引擎是底座不是出口**：它提供证据，认知引擎负责转化
