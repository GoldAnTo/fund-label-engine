# 项目边界

## 项目内

- 基金数据标准化输入。
- 数据覆盖率检查。
- 特征计算。
- 规则标签计算。
- 标签证据生成。
- 标签批次记录。
- 人工复核数据模型。
- FastAPI 服务骨架。
- 后续标签工作台预留。

## 项目外

- 自动交易。
- 实盘委托。
- 投资建议或买卖推荐。
- 全市场所有基金类型一次性覆盖。
- LLM 自动决定基金准入。
- 无证据的主观标签。
- 直接商用第三方开源项目代码。

## 第一版基金范围

纳入：

- 股票型
- 混合型-偏股
- 混合型-灵活
- 指数型-股票

暂不纳入正式标签：

- 纯债基金
- 货币基金
- QDII
- FOF
- 商品基金
- REITs

这些类型后续可以单独设计标签体系，不能套用权益基金标签。

## 高级标签边界

以下标签必须等 `stock_factors` 或 `stock_labels` 数据具备后才能输出正式结果：

- 深度价值
- 质量成长
- 红利稳健
- 低估修复
- 高估高波动
- 风格漂移

在缺少股票因子时，只能输出：

- `style_unlabeled_stock_factors_missing`
- `manual_review_required`
- `observe_only`
