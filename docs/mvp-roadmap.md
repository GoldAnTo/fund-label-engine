# MVP 路线图

## Phase 0: 项目启动（已完成）

- 建立项目目录。
- 写清楚需求、边界、基准项目和数据契约。
- 起最小 Python 包和测试。
- 实现单只基金标签计算闭环。

## Phase 1: 本地规则引擎（已完成第一批）

- 支持结构化 `FundInput`。
- 支持覆盖率检查。
- 支持基础特征计算。
- 支持第一批标签。
- 输出标签证据。
- 支持规则阈值配置。
- 支持数据不足时输出可观察标签。

## Phase 2: 数据接入（真实数据闭环已完成）

- 接入 `fundData` 本地 SQLite 或导出的 JSON。
- 支持批量读取基金样本。
- 记录 `label_runs`。
- 保存 `fund_label_results` 和 `fund_label_evidence`。
- 保存 `fund_run_coverage` 和 `label_definitions`。
- 股票因子按持仓报告期做 as-of 对齐。
- 生成基金级 `fund_factor_exposures`，供标签引擎优先消费。
- 支持相对基准组件解析、`benchmark_components` 审计和本地 `benchmark_component_returns` 稳定源入口。

## Phase 3: API 服务（基础查询和复核已完成）

- FastAPI 暴露 `/health`。
- FastAPI 暴露 `/v1/funds/{fund_code}/labels`。
- FastAPI 暴露 `/v1/runs`。
- 支持按批次查询结果。
- 支持写入和查询人工复核记录。

## Phase 4: 工作台（基础完成，增强中）

- 展示基金标签。
- 展示证据链。
- 支持人工确认、驳回、观察。
- 支持导出审核报告。
- 支持按标签、复核动作、业务池、分组类型和分类检索基金。
- 下一步增强：把分类/分组分布做成可点击入口，并展示基准组件缺口。

## Phase 5: 股票因子和高级风格（主线完成，继续校准）

- 建立 `stock_factors`。
- 做持仓穿透。
- 计算价值、成长、红利、质量、行业主题暴露。
- 评估风格稳定性和风格漂移。
- 对低覆盖风格暴露加质量门槛，避免误出正式风格标签。
- 继续积累多期报告期样本，再决定是否把风格稳定性从观察口径升级。

## 近期推荐顺序

1. 补稳定基准组件收益源，优先债券指数、行业/主题指数和港股指数。
2. 用 `benchmark_components` unresolved 榜单驱动数据补齐，而不是继续增加模糊映射。
3. 在工作台突出业务池、基准缺口和风格稳定性证据。
4. 做规则回放和标签启停，方便阈值校准。
5. 继续按真实样本复核收益风险、费率和风格口径。
